نیاز به سیستم کنترل دسترسی مدرن

کتابخانه‌های دانشگاهی و عمومی با چالش‌های متعددی در مدیریت تردد روبرو هستند. کارت‌های سنتی اغلب گم می‌شوند، کپی می‌شوند یا فراموش می‌شوند. نیاز به سیستمی بود که:

  • بدون نیاز به کارت یا دستگاه فیزیکی کار کند
  • امن و قابل اعتماد باشد
  • تجربه کاربری ساده‌ای داشته باشد
  • هزینه پیاده‌سازی مناسبی داشته باشد

راهکار ما: استفاده از gesture recognition برای کنترل دسترسی. کاربران با ژست خاصی (مانند دست دادن) می‌توانند درب را باز کنند.

معماری سیستم

سیستم از سه بخش اصلی تشکیل شده:

  1. Hardware: دوربین، Raspberry Pi، قفل الکترونیکی
  2. Computer Vision: تشخیص ژست با MediaPipe
  3. Software: رابط مدیریتی با PyQt6

انتخاب تکنولوژی تشخیص ژست

از بین گزینه‌های موجود، MediaPipe را انتخاب کردیم چون:

  • دقت بالا در تشخیص ژست‌های دست
  • سرعت مناسب برای real-time processing
  • راحتی پیاده‌سازی و یکپارچه‌سازی
  • عدم نیاز به GPU قدرتمند

MediaPipe Hand Tracking از ۲۱ نقطه کلیدی روی دست استفاده می‌کند که امکان تشخیص ژست‌های پیچیده را فراهم می‌کند.

پیاده‌سازی Gesture Recognition

ژست‌های تعریف شده

سه ژست اصلی برای کنترل دسترسی تعریف کردیم:

  • Peace Sign (V): برای باز کردن درب ورودی
  • Thumbs Up: برای تایید حضور
  • Open Palm: برای لغو عملیات

هر ژست با ترکیب وضعیت انگشتان تعریف می‌شود. مثلاً برای peace sign، انگشت اشاره و میانی باید باز باشند و سایر انگشتان بسته.

الگوریتم تشخیص ژست

برای تشخیص ژست از heuristic rules استفاده کردیم:

  1. تشخیص نقاط کلیدی دست با MediaPipe
  2. محاسبه زاویه بین مفاصل انگشتان
  3. بررسی وضعیت باز/بسته بودن هر انگشت
  4. مقایسه با الگوهای تعریف شده
  5. اعمال فیلتر زمانی برای جلوگیری از false positive

مدیریت کاربران

سیستم از دو سطح دسترسی پشتیبانی می‌کند:

  • کاربران عادی: با ژست peace sign وارد می‌شوند
  • مدیران: علاوه بر ژست، نیاز به تشخیص چهره دارند

برای امنیت بیشتر، ژست باید به مدت ۲ ثانیه نگه داشته شود تا از accidental activation جلوگیری شود.

چالش‌های پیاده‌سازی

چالش ۱: شرایط نوری متغیر

کتابخانه‌ها شرایط نوری متفاوتی دارند. برای اطمینان از عملکرد پایدار:

  • استفاده از دوربین با قابلیت night vision
  • تنظیم automatic brightness و contrast
  • آموزش مدل با داده‌های متنوع نوری

چالش ۲: تشخیص ژست در فواصل مختلف

کاربران در فاصله‌های متفاوتی از دوربین قرار می‌گیرند. برای حل این مشکل:

  • استفاده از bounding box برای normalization اندازه دست
  • Dynamic threshold بر اساس فاصله
  • Multi-scale detection

چالش ۳: جلوگیری از spoofing

سیستم باید در برابر تلاش‌های spoofing مقاوم باشد:

  • تشخیص حرکت واقعی دست (نه تصویر ثابت)
  • بررسی continuity ژست در طول زمان
  • Liveness detection برای اطمینان از حضور واقعی کاربر

چالش ۴: تجربه کاربری

کاربران باید به راحتی یاد بگیرند چگونه از سیستم استفاده کنند:

  • نمایش visual feedback در لحظه
  • آموزش تعاملی برای کاربران جدید
  • پیام‌های خطا واضح و مفید

رابط کاربری PyQt6

اپلیکیشن دسکتاپ برای مدیریت سیستم شامل قابلیت‌های زیر:

  • Live Camera Feed: نمایش تصویر دوربین به صورت real-time
  • Gesture Recognition Display: نمایش ژست تشخیص داده شده
  • Access Logs: ثبت تمام ترددهای موفق و ناموفق
  • User Management: اضافه/حذف کاربران مجاز
  • Settings: تنظیم پارامترهای تشخیص

ویژگی‌های پیشرفته

  • Multi-camera Support: پشتیبانی از چند دوربین همزمان
  • Time-based Access: کنترل دسترسی بر اساس زمان
  • Integration with Library System: اتصال به سیستم کتابخانه
  • Mobile App: مدیریت از طریق اپ موبایل

بهینه‌سازی عملکرد

بهینه‌سازی MediaPipe

برای بهبود عملکرد real-time:

  • کاهش resolution ورودی به 320x240
  • استفاده از model complexity پایین
  • Parallel processing برای چند دست

مدیریت منابع سیستم

Raspberry Pi منابع محدودی دارد، بنابراین:

  • استفاده از threading برای پردازش موازی
  • Memory pooling برای کاهش allocation
  • Background processing برای عملیات غیرضروری

امنیت و قابلیت اطمینان

امنیت فیزیکی

  • قفل الکترونیکی با backup مکانیکی
  • تشخیص tampering با دوربین
  • Alarm system برای تلاش‌های غیرمجاز

امنیت دیجیتال

  • Encryption داده‌های کاربران
  • Secure communication با سرور مرکزی
  • Regular security updates
